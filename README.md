## Problem 3 — What to change

**1. Set DATA_DIR** at the top of `problem3.py`:
```python
DATA_DIR = "."   # Change to the folder with your CSV files
```

**2. If column names differ**, update the column references.
The scripts expect:
- Trades: `timestamp`, `trade_id`, `wallet_id`, `price`, `quantity`, `side`
- Market: `timestamp`, `open`, `high`, `low`, `close`, `volume`

**3. Two-pass strategy** — after running, review `submission.csv` and manually
remove any rows you are not confident about. Each false positive costs −2 points.

## Violation Type Reference (Problem 3)

| Violation Type           | Where to find it         | Key signal                              |
|--------------------------|--------------------------|------------------------------------------|
| `peg_break`              | USDCUSDT                 | price ≠ $1.00 by > 0.5%                |
| `wash_volume_at_peg`     | USDCUSDT                 | same wallet buys + sells at $1.00       |
| `aml_structuring`        | BATUSDT, XRPUSDT, LTCUSDT | trades just below round thresholds     |
| `coordinated_structuring`| BATUSDT                  | multiple wallets, same pattern, same day|
| `threshold_testing`      | BATUSDT                  | one trade AT threshold, then many below |
| `pump_and_dump`          | DOGEUSDT, SOLUSDT        | 5+ rising bars then sharp crash         |
| `coordinated_pump`       | DOGEUSDT                 | 3+ wallets buying during pump           |
| `wash_trading`           | LTCUSDT, BTC, ETH        | same wallet, near-zero net position     |
| `round_trip_wash`        | LTCUSDT                  | two wallets mirror each other           |
| `ramping`                | LTCUSDT, XRPUSDT         | monotonically rising prices, same wallet|
| `layering_echo`          | ETHUSDT, BTCUSDT         | many buys then immediate reversal       |

## Approach Summary

**Problem 3:**
1. Start with USDCUSDT (peg break — 1 line of code, free points)
2. Move to BATUSDT (AML structuring — look for uniform trade sizes below round thresholds)
3. DOGEUSDT / SOLUSDT (pump-and-dump — look for 5+ bar price run then crash)
4. LTCUSDT (wash trading — near-zero net position per wallet)
5. XRPUSDT (AML + wash)
6. ETHUSDT / BTCUSDT (Isolation Forest + layering)

**Problem 1:**
1. Compute OBI, spread_bps, level-1 concentration per ticker per minute
2. Rolling z-scores against each ticker's own recent history
3. DBSCAN clustering of anomalous windows
4. Generate alerts for windows where ≥2 features are simultaneously extreme

**Problem 2:**
1. Scrape EDGAR for 8-K filings across all tickers
2. Compute 15-day rolling volume baseline per ticker
3. Flag tickers where volume z-score > 3 in the 5 days before an announcement
4. Find the specific trader making large pre-announcement buys
